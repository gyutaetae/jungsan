# User Flow

## Primary Flow: Receipt Images

1. 사용자가 단일 화면에 진입한다.
2. 동일 비중의 입력 카드 2개를 본다: `영수증 촬영/업로드`, `거래내역 엑셀 업로드`.
3. 사용자가 `영수증 촬영/업로드`를 누른다.
4. 모바일은 기본 카메라/사진 선택, 데스크톱은 파일 선택을 연다.
5. 사용자가 여러 이미지를 선택한다.
6. 앱은 처리 큐에 이미지를 추가한다.
7. 앱은 이미지를 1장씩 `/api/extract-receipt`로 보낸다.
8. 성공 시 장부 행을 검토 테이블에 추가한다.
9. 실패 시 해당 항목에 `실패`를 표시하고 `샘플 결과 사용`을 제공한다.
10. 사용자는 검토 테이블에서 값을 수정한다.
11. 필요경비 요약이 실시간 갱신된다.
12. 사용자가 `총수입금액`을 입력한다.
13. 앱은 홈택스 입력용 요약을 갱신한다.
14. 사용자가 `간편장부 엑셀 다운로드`를 누른다.
15. 확인 필요 항목이 있으면 경고를 표시하되 다운로드는 허용한다.
16. 앱은 `/api/export-ledger`로 장부 행을 보내 엑셀 파일을 받는다.
17. 사용자가 `신고 준비 문서 다운로드`를 누른다.
18. 앱은 `/api/export-tax-prep`로 총수입금액과 장부 행을 보내 검토용 xlsx 초안을 받는다.
19. 사용자는 xlsx의 값을 홈택스 입력 전 확인한다.

## Secondary Flow: Transaction Spreadsheet

1. 사용자가 `거래내역 엑셀 업로드`를 누른다.
2. `.xlsx` 또는 `.csv` 파일을 선택한다.
3. 앱은 `/api/import-transactions`로 파일을 보낸다.
4. 서버는 샘플 포맷 기준으로 행을 정규화한다.
5. 결과는 기존 검토 테이블에 병합된다.
6. 이후 수정, 요약, 다운로드 흐름은 영수증과 동일하다.

## Camera Batch Flow

1. 휴대폰 사용자가 `영수증 촬영/업로드` 카드를 누른다.
2. 앱은 `기존 사진 선택`과 `카메라로 여러 장 촬영` 선택지를 보여준다.
3. 사용자가 `카메라로 여러 장 촬영`을 선택하면 브라우저 카메라 권한을 요청하고 프리뷰를 표시한다.
4. 사용자가 영수증을 한 장씩 촬영한다.
5. 앱은 촬영 이미지를 서버에 저장하지 않고 클라이언트 임시 목록에 추가한다.
6. 사용자는 임시 목록에서 잘못 찍은 이미지를 삭제하거나 계속 촬영한다.
7. 사용자가 `한 번에 장부 초안 만들기`를 누른다.
8. 앱은 임시 목록의 이미지를 기존 처리 큐에 넘긴다.
9. 앱은 이미지를 1장씩 `/api/extract-receipt`로 보내 분석한다.
10. 성공한 결과는 기존 검토 테이블에 추가된다.
11. 실패한 항목은 기존 영수증 처리와 같이 `실패`를 표시하고 `샘플 결과 사용`을 제공한다.
12. 이후 수정, 요약, 다운로드 흐름은 영수증 업로드와 동일하다.

## Tax Prep Flow

1. 사용자가 검토 테이블의 비용 항목을 확인하고 수정한다.
2. 사용자가 요약 패널에서 `총수입금액`을 직접 입력한다.
3. 앱은 총수입금액, 필요경비 합계, 예상 소득금액, 계정과목별 합계, 증빙유형별 합계, 확인 필요 항목 수를 표시한다.
4. 사용자가 `신고 준비 문서 다운로드`를 누른다.
5. 앱은 `jeongsan-tax-prep-draft.xlsx`를 다운로드한다.
6. 파일에는 홈택스 입력용 요약과 공식 HWP 항목 구조를 참고한 신고 준비 문서 초안이 포함된다.
7. 사용자는 홈택스 입력 전 금액과 확인 필요 항목을 검토한다.

## Empty State

- 메시지: `아직 장부에 들어간 항목이 없어요`
- 보조 메시지: `영수증 사진이나 거래내역 엑셀을 넣어보세요`
- 액션: `샘플 데이터로 보기`

## Processing States

- `queued`: 대기
- `processing`: 읽는 중
- `needs_review`: 확인 필요
- `confirmed`: 완료
- `failed`: 실패

## Download Rule

확인 필요 항목이 있어도 다운로드는 가능하다. 단, 다음 경고를 먼저 보여준다.

`확인 필요 항목이 있어요. 그래도 장부를 만들까요?`

Actions: `검토 계속`, `그래도 만들기`

## Wording Rule

- 신고 준비 산출물은 `검토용 초안` 또는 `신고 준비 문서 초안`으로 부른다.
- 최종 신고서류처럼 표현하지 않는다.
- 다운로드 전후 문구에는 `홈택스 입력 전 확인 필요`를 명확히 표시한다.
