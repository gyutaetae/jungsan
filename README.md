# 정산

영수증 사진과 거래내역 엑셀을 넣으면, 검토 가능한 장부 초안과 홈택스 입력용 요약을 만들어주는 AI 정산 작업대입니다.

`정산`은 홈택스 자동 제출 서비스가 아닙니다. 소상공인과 프리랜서가 신고 전에 가장 오래 붙잡히는 자료 정리, 필요경비 계산, 장부 초안 작성, 입력값 확인을 줄이는 서비스입니다.

## 한 줄 피치

> 흩어진 영수증과 거래내역을 검토 가능한 장부와 신고 준비 요약으로 바꿔, 소상공인의 세금 신고 준비 시간을 줄입니다.

## 문제

개인 소상공인과 프리랜서에게 세금 신고가 어려운 이유는 신고 버튼을 누르는 순간만이 아닙니다.

신고 전에 먼저 해야 하는 일이 많습니다.

- 사진첩에 흩어진 영수증을 모아야 합니다.
- 카드사, 은행, 플랫폼 거래내역을 따로 확인해야 합니다.
- 날짜, 거래처, 금액, 공급가액, 부가세, 증빙유형을 장부 형식에 맞게 옮겨야 합니다.
- 홈택스 입력 전에 필요경비와 소득금액을 다시 계산해야 합니다.

이 과정은 전문 세무 지식보다 반복 입력과 확인 작업이 많습니다. 그래서 세무사를 쓰더라도 사용자는 결국 자료를 모아 넘겨야 하고, 직접 신고를 준비하는 사람은 장부 초안부터 막힙니다.

## 해결

`정산`은 신고 전 자료 정리를 하나의 작업 흐름으로 묶습니다.

1. 영수증 사진 또는 거래내역 엑셀을 업로드합니다.
2. AI와 파일 파서가 거래일, 거래처, 금액, 세금값, 증빙유형을 장부 행으로 정리합니다.
3. 사용자는 카드형 검토 화면에서 맞는 항목만 `확인`하고, 틀린 값만 고칩니다.
4. 총수입금액을 입력하면 필요경비, 예상 소득금액, 홈택스 입력용 요약이 계산됩니다.
5. 간편장부 엑셀과 신고 준비 문서 xlsx 초안을 다운로드합니다.

핵심은 “AI가 대신 신고한다”가 아니라 “AI가 먼저 정리하고, 사람은 검토한다”입니다.

## 왜 이 접근이 현실적인가

홈택스 자동 로그인, 인증서 처리, 자동 제출은 보안, 법적 책임, 세무 판단 문제가 큽니다. 아이디어톤 MVP에서 이 영역을 무리하게 자동화하면 데모는 화려해 보여도 실제 신뢰도가 낮아집니다.

`정산`은 자동 제출보다 앞단의 문제를 해결합니다.

- 사용자가 실제로 막히는 지점은 자료 정리와 입력값 준비입니다.
- 간편장부는 최종 목적이 아니라 홈택스 신고 준비의 기반 데이터입니다.
- 확실하지 않은 세금값은 억지로 추정하지 않고 확인 필요로 남깁니다.
- 모든 결과는 사용자가 직접 보고 수정할 수 있습니다.

그래서 `정산`은 세무 판단을 대체하는 서비스가 아니라, 신고 준비를 빠르게 시작하게 만드는 검토 가능한 작업대입니다.

## 타깃 사용자

- 혼자 장부를 정리하는 개인 소상공인
- 영수증과 카드 지출이 자주 생기는 프리랜서
- 세무사에게 넘길 자료를 매번 수작업으로 모으는 사용자
- 홈택스와 장부 양식은 부담스럽지만 지출 정리는 직접 해야 하는 사용자

## MVP 기능

- 영수증 촬영 및 이미지 업로드
- 여러 영수증 선택 후 1장씩 순차 분석
- 거래내역 `.xlsx` / `.csv` 업로드
- 업로드 결과를 하나의 장부 데이터로 병합
- 카드형 검토 UI에서 거래일, 거래처, 금액, 공급가액, 부가세, 계정과목, 증빙유형 수정
- 확인 버튼 기반 검토 완료 처리
- 금액 입력 세 자리 콤마 표시
- 필요경비 요약 표시
- 총수입금액 입력 기반 홈택스 입력용 요약 표시
- 간편장부 엑셀 다운로드
- 신고 준비 문서 xlsx 초안 다운로드
- `총수입금액 및 필요경비명세서` 초안 생성
- `간편장부소득금액계산서` 초안 생성
- 다크모드 및 한국어/영어 전환
- AI 실패 상황에서도 시연 가능한 샘플 결과 fallback

## 데모 흐름

1. `샘플 데이터로 보기`를 눌러 즉시 장부 초안을 확인합니다.
2. 또는 영수증 이미지와 거래내역 엑셀을 업로드합니다.
3. 각 거래 카드에서 값이 맞으면 `확인`을 누르고, 틀리면 직접 수정합니다.
4. 필요경비 요약에서 비용 합계와 확인 필요 항목을 봅니다.
5. 총수입금액을 입력해 홈택스 입력용 요약을 확인합니다.
6. `간편장부 엑셀 다운로드`로 장부 초안을 받습니다.
7. `신고 준비 문서 다운로드`로 신고 전 확인용 xlsx 초안을 받습니다.

## 산출물

### 간편장부 엑셀

검토된 거래일, 거래처, 내용, 총액, 공급가액, 부가세, 계정과목, 증빙유형을 바탕으로 `jeongsan-simple-ledger.xlsx`를 생성합니다.

### 신고 준비 문서 초안

총수입금액과 검토된 장부 항목을 바탕으로 `jeongsan-tax-prep-draft.xlsx`를 생성합니다.

포함 시트:

- `홈택스 입력용 요약`
- `총수입금액 및 필요경비명세서`
- `간편장부소득금액계산서`
- `확인 필요 항목`

국세청 공식 HWP 서식은 기준 문서로 보고, MVP에서는 공식 항목 구조를 참고한 xlsx 검토용 초안을 생성합니다. 세무조정 항목은 자동 계산하지 않으며, 신고 전 사용자의 확인이 필요합니다.

## 심사 포인트

### 1. 문제의 현실성

소상공인과 프리랜서에게 세금 신고는 1년에 한 번의 이벤트가 아니라, 영수증과 거래내역을 계속 모아야 하는 반복 업무입니다. `정산`은 이 반복 업무를 줄입니다.

### 2. 신뢰 가능한 AI 사용

AI가 모든 값을 단정하지 않습니다. 모르는 값은 비워두고 확인 필요로 남기며, 사용자가 검토 화면에서 직접 확정합니다.

### 3. 데모 완성도

단순 OCR 결과가 아니라 장부 초안, 필요경비 요약, 홈택스 입력용 요약, 신고 준비 문서 다운로드까지 이어지는 end-to-end 흐름을 보여줍니다.

### 4. 확장 가능성

현재 MVP는 신고 준비 자동화에 집중합니다. 이후에는 홈택스 입력 화면별 체크리스트, 카드사/플랫폼 정산서 업로드, 세무사 공유 링크, 사용자 승인 기반 입력 보조로 확장할 수 있습니다.

## 기술 스택

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- OpenAI Responses API
- zod
- xlsx
- exceljs
- Vercel

서버 데이터베이스, 로그인, 업로드 원본 저장은 사용하지 않습니다. API route는 업로드 파일을 일시 처리한 뒤 장부 행 또는 엑셀 파일만 반환합니다.

## 발표 메시지

`정산`은 세금 신고를 대신하는 서비스가 아닙니다.

대신 신고 전에 사용자가 가장 먼저 막히는 일을 해결합니다. 흩어진 영수증과 거래내역을 장부 초안으로 만들고, 사람이 검토한 값만 신고 준비 요약과 문서 초안으로 바꿉니다.

아이디어톤에서 보여줄 핵심은 단순 자동화가 아니라, 소상공인이 믿고 사용할 수 있는 “검토 가능한 AI 정산 흐름”입니다.
