# AI로 영수증지옥을 없앤다면

[서비스 주소](https://jungsan.vercel.app)

소상공인과 프리랜서가 영수증 사진과 거래내역 엑셀을 넣으면, 신고 준비에 필요한 장부 초안과 홈택스 입력용 요약을 만들어주는 AI 신고 준비 비서입니다.

> 혼자 매출과 비용을 챙겨야 하는 소상공인·프리랜서를 위해, 흩어진 지출 자료를 검토 가능한 장부와 신고 준비 문서 초안으로 바꿔 세금 신고 전 가장 번거로운 정리 시간을 줄입니다.

## 무엇을 해결하려고 하는가

소상공인, 프리랜서, 작은 팀에게 정산이 어려운 이유는 계산 자체보다 자료 정리입니다.

- 영수증은 사진첩과 메신저에 흩어져 있습니다.
- 카드사와 플랫폼 거래내역은 각자 다른 엑셀 파일로 내려옵니다.
- 장부에는 날짜, 거래처, 금액, 공급가액, 부가세, 계정과목, 증빙유형을 다시 맞춰 넣어야 합니다.
- 신고나 비용 정산 직전에는 같은 숫자를 다시 확인하고 다시 옮겨 적어야 합니다.

이 과정은 어렵다기보다 지칩니다. 그래서 `정산`은 영수증과 거래내역을 먼저 읽고, 사람이 확인할 수 있는 거래 카드와 요약으로 바꾸는 데 집중합니다.

## 사용 흐름

1. 영수증 이미지나 거래내역 엑셀을 업로드합니다.
2. AI와 파일 파서가 거래 정보를 읽어 카드 형태로 정리합니다.
3. 사용자는 금액, 계정과목, 증빙유형처럼 중요한 값만 확인합니다.
4. 확인된 거래만 필요경비 요약과 소득금액 계산에 반영됩니다.
5. 간편장부 엑셀과 신고 준비용 xlsx 초안을 다운로드합니다.

핵심은 AI가 모든 것을 확정하는 것이 아닙니다. AI가 먼저 정리하고, 사용자는 틀린 부분만 고칩니다. 세금이나 정산처럼 틀리면 곤란한 영역에서는 이 확인 과정이 꼭 필요하다고 봤습니다.

## 주요 기능

- 영수증 이미지 인식 및 거래내역 엑셀 업로드
- 카메라로 여러 장 촬영 후 한 번에 장부 초안 생성
- 거래일, 거래처, 금액, 공급가액, 부가세, 계정과목, 증빙유형 추출
- 모바일에서도 보기 쉬운 카드형 검토 UI
- `확인` 버튼 기반 검토 완료 처리
- 필요경비 요약 및 총수입금액 기반 소득금액 계산
- 홈택스 입력용 요약 화면
- 간편장부 엑셀 다운로드
- 신고 준비 문서 xlsx 초안 생성

## 왜 자동 제출까지 하지 않는가

`정산`은 홈택스 자동 제출 서비스가 아닙니다. 세금 신고 자동 제출은 인증, 보안, 법적 책임, 세무 판단 리스크가 큽니다.

대신 지금 가장 먼저 줄일 수 있는 부분에 집중했습니다. 사용자가 신고 전에 오래 붙잡히는 영수증 정리, 비용 분류, 장부 초안 작성, 입력값 준비를 덜어주는 것입니다.

그래서 이 서비스는 다음 원칙을 가집니다.

- 모르는 값은 억지로 채우지 않습니다.
- 사용자가 확인한 거래만 완료 처리합니다.
- 산출물은 제출본이 아니라 신고 전 검토용 초안입니다.
- 기본 데모에서는 서버 데이터베이스와 로그인을 쓰지 않고 원본 파일을 저장하지 않습니다.
- 수업용 확장안에서는 Supabase Postgres로 장부 행과 신고 준비 요약만 저장하고, 원본 이미지는 선택 저장으로 둡니다.

## 산출물

### 간편장부 엑셀

확인된 거래의 거래일, 거래처, 내용, 총액, 공급가액, 부가세, 계정과목, 증빙유형을 바탕으로 `jeongsan-simple-ledger.xlsx`를 생성합니다.

### 신고 준비 문서 초안

총수입금액과 확인된 비용 항목을 바탕으로 `jeongsan-tax-prep-draft.xlsx`를 생성합니다.

포함 시트:

- 홈택스 입력용 요약
- 총수입금액 및 필요경비명세서
- 간편장부소득금액계산서
- 확인 필요 항목

### 1. 실제 문제를 겨냥합니다

소상공인과 프리랜서는 신고 버튼보다 신고 전 자료 정리에서 먼저 막힙니다. `정산`은 이 반복 업무를 줄입니다.

### 2. AI를 믿을 수 있게 씁니다

AI가 모든 값을 단정하지 않습니다. 검토 카드에서 사용자가 직접 확인하고, 확인된 값만 산출물에 반영합니다.

### 3. 데모가 끝까지 이어집니다

업로드에서 끝나지 않고 장부 카드, 필요경비 요약, 홈택스 입력용 요약, 문서 다운로드까지 이어지는 end-to-end 흐름을 보여줍니다.

### 4. 데이터베이스 설계가 설명 가능합니다

Supabase Postgres 확장 스키마로 사용자, 신고 단위, 업로드 묶음, 파일 메타데이터, 장부 행, 신고 준비 요약을 분리했습니다. 정규화, 제약조건, 인덱스, RLS, 집계 뷰를 수업 평가 포인트로 보여줄 수 있습니다.
