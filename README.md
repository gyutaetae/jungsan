# AI로 영수증지옥을 없앤다면

[서비스 주소](https://jungsan.vercel.app)

영수증 정리는 이상하게 늘 마지막에 남습니다. 결제는 몇 초면 끝나는데, 나중에 정산하거나 신고를 준비하려고 보면 사진첩, 카드 내역, 플랫폼 정산서, 엑셀 파일이 전부 따로 흩어져 있습니다. 결국 날짜와 거래처, 금액, 증빙유형을 다시 보고 다시 옮겨 적게 됩니다.

`정산`은 이 반복을 줄이려고 만든 서비스입니다. 아이디어를 보여주기 위해 만든 일회성 데모라기보다, 실제로 영수증과 거래내역을 정리해야 하는 사람이 덜 지치도록 만드는 작업대에 가깝습니다.

## 무엇을 해결하려고 하는가

소상공인, 프리랜서, 작은 팀에게 정산이 어려운 이유는 계산 자체보다 자료 정리입니다.

- 영수증은 사진첩과 메신저에 흩어져 있습니다.
- 카드사와 플랫폼 거래내역은 각자 다른 엑셀 파일로 내려옵니다.
- 장부에는 날짜, 거래처, 금액, 공급가액, 부가세, 계정과목, 증빙유형을 다시 맞춰 넣어야 합니다.
- 신고나 비용 정산 직전에는 같은 숫자를 다시 확인하고 다시 옮겨 적어야 합니다.

이 과정은 어렵다기보다 지칩니다. 그래서 `정산`은 영수증과 거래내역을 먼저 읽고, 사람이 확인할 수 있는 거래 카드와 요약으로 바꾸는 데 집중합니다.

## 사용 흐름

1. 영수증 이미지나 거래내역 엑셀을 업로드합니다.
2. AI와 파일 파서가 거래 정보를 읽어 카드 형태로 정리합니다.
3. 사용자는 금액, 계정과목, 증빙유형처럼 중요한 값만 확인합니다.
4. 확인된 거래만 필요경비 요약과 소득금액 계산에 반영됩니다.
5. 간편장부 엑셀과 신고 준비용 xlsx 초안을 다운로드합니다.

핵심은 AI가 모든 것을 확정하는 것이 아닙니다. AI가 먼저 정리하고, 사용자는 틀린 부분만 고칩니다. 세금이나 정산처럼 틀리면 곤란한 영역에서는 이 확인 과정이 꼭 필요하다고 봤습니다.

## 주요 기능

- 영수증 이미지 인식
- 거래내역 엑셀 업로드
- 거래일, 거래처, 금액, 공급가액, 부가세 추출
- 계정과목과 증빙유형 정리
- 모바일에서도 확인하기 쉬운 거래 카드 UI
- 확인된 거래만 반영하는 검토 흐름
- 필요경비 요약과 예상 소득금액 계산
- 홈택스 입력에 참고할 수 있는 요약 화면
- 간편장부 엑셀 다운로드
- 신고 준비 문서 xlsx 초안 생성

## 왜 자동 제출까지 하지 않는가

`정산`은 홈택스 자동 제출 서비스가 아닙니다. 세금 신고 자동 제출은 인증, 보안, 법적 책임, 세무 판단 리스크가 큽니다.

대신 지금 가장 먼저 줄일 수 있는 부분에 집중했습니다. 사용자가 신고 전에 오래 붙잡히는 영수증 정리, 비용 분류, 장부 초안 작성, 입력값 준비를 덜어주는 것입니다.

그래서 이 서비스는 다음 원칙을 가집니다.

- 모르는 값은 억지로 채우지 않습니다.
- 사용자가 확인한 거래만 완료 처리합니다.
- 결과물은 제출본이 아니라 검토용 초안입니다.
- 원본 파일을 서버에 오래 저장하는 구조를 전제로 하지 않습니다.

## 산출물

### 간편장부 엑셀

확인된 거래의 거래일, 거래처, 내용, 총액, 공급가액, 부가세, 계정과목, 증빙유형을 바탕으로 `jeongsan-simple-ledger.xlsx`를 생성합니다.

### 신고 준비 문서 초안

총수입금액과 확인된 비용 항목을 바탕으로 `jeongsan-tax-prep-draft.xlsx`를 생성합니다.

포함 시트:

- 홈택스 입력용 요약
- 총수입금액 및 필요경비명세서
- 간편장부소득금액계산서
- 확인 필요 항목

## 실행 방법

```bash
npm install
npm run dev
```

AI 영수증 추출을 사용하려면 환경변수에 사용할 모델 제공자의 API 키를 설정해야 합니다.

```env
OPENAI_API_KEY=
ANTHROPIC_API_KEY=
```

## 한 문장으로

`정산`은 신고를 대신하는 서비스가 아닙니다. 영수증과 거래내역을 사람이 검토할 수 있는 장부 초안과 입력값으로 바꿔, 신고 전 가장 오래 걸리는 정리 시간을 줄이는 서비스입니다.
