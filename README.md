# 정산

영수증 사진과 거래내역 엑셀을 넣으면, 개인 소상공인과 프리랜서가 바로 검토할 수 있는 간편장부 초안, 홈택스 입력용 요약, 신고 준비 문서 검토용 초안을 만들어주는 AI 정산 비서입니다.

## 문제

소상공인과 프리랜서에게 세금 신고가 어려운 이유는 신고 버튼을 누르는 순간만이 아닙니다. 그 전에 영수증을 모으고, 카드 내역을 정리하고, 장부 양식에 맞게 옮겨 적는 반복 작업이 먼저 필요합니다.

영수증은 사진첩에 흩어져 있고, 카드 내역은 엑셀로 따로 있고, 홈택스나 세무 신고에 필요한 장부 형식은 또 따로 있습니다. 결국 사용자는 매번 날짜, 거래처, 금액, 공급가액, 부가세, 증빙유형을 직접 확인하며 장부를 만들어야 합니다.

`정산`은 이 가장 번거로운 앞단 작업을 줄입니다.

## 해결

`정산`은 영수증과 거래내역을 하나의 장부 초안으로 모아줍니다.

1. 사용자가 영수증을 촬영하거나 이미지를 업로드합니다.
2. 거래내역 엑셀 또는 CSV 파일도 함께 올릴 수 있습니다.
3. AI가 날짜, 거래처, 금액, 세금값, 증빙유형을 읽어 장부 행으로 정리합니다.
4. 사용자는 검토 테이블에서 틀린 값만 고칩니다.
5. 필요경비 요약과 홈택스 입력용 요약을 확인합니다.
6. 간편장부 엑셀과 신고 준비 문서 검토용 초안을 다운로드합니다.

핵심은 완전 자동 신고가 아닙니다. AI가 먼저 정리하고, 사람은 중요한 부분만 확인하는 구조입니다.

## 왜 간편장부인가

개인 소상공인과 프리랜서에게 가장 현실적으로 필요한 산출물은 바로 세무 신고 전에 정리된 장부입니다. 세무사를 쓰더라도 자료를 모아서 넘기는 일은 본인이 해야 하고, 직접 신고를 준비한다면 장부 초안은 더 중요합니다.

그래서 `정산`은 홈택스 자동 제출보다 먼저 필요한 문제를 해결합니다.

흩어진 지출 자료를 검토 가능한 간편장부 초안으로 바꾸고, 사용자가 홈택스 입력 전 확인할 요약과 신고 준비 문서 초안을 내려받아 세무 신고 준비나 세무사 전달에 활용할 수 있게 합니다.

## 신뢰 설계

세금과 돈 문제에서 AI가 모든 값을 자신 있게 채우는 것은 오히려 위험합니다.

`정산`은 확실하지 않은 공급가액, 부가세 같은 세금값을 억지로 추정하지 않습니다. 모르는 값은 비워두고 `확인 필요`로 표시합니다. 사용자는 AI 결과를 그대로 믿는 것이 아니라, 검토 테이블에서 직접 확인하고 수정합니다.

이 방식은 데모에서도 명확합니다. `정산`은 세무 판단을 대신하는 AI가 아니라, 장부의 첫 초안을 빠르게 만들어주는 작업대입니다.

## 타깃 사용자

- 혼자 장부를 정리하는 개인 소상공인
- 영수증과 카드 지출이 자주 생기는 프리랜서
- 세무사에게 넘길 자료를 매번 수작업으로 모으는 사용자
- 홈택스와 장부 양식은 부담스럽지만 지출 정리는 직접 해야 하는 사용자

## MVP 기능

- 영수증 촬영 및 이미지 업로드
- 여러 영수증 선택 후 1장씩 순차 분석
- 거래내역 `.xlsx` / `.csv` 업로드
- 모든 입력을 하나의 장부 검토 테이블로 병합
- 거래일, 거래처, 금액, 공급가액, 부가세, 계정과목, 증빙유형 직접 수정
- 필요경비 요약 표시
- 총수입금액 입력 기반 홈택스 입력용 요약 표시
- 확인 필요 항목 경고 후 간편장부 엑셀 다운로드
- 신고 준비 문서 검토용 xlsx 초안 다운로드
- AI 실패 상황에서도 시연 가능한 샘플 결과 fallback

## 데모 흐름

1. 첫 화면에서 `샘플 데이터로 보기`를 눌러 검토 테이블과 요약 패널을 즉시 확인합니다.
2. 또는 `영수증 촬영/업로드`로 이미지를 넣어 AI 추출 결과를 장부 행으로 추가합니다.
3. `거래내역 엑셀 업로드`로 `.xlsx` 또는 `.csv` 파일을 추가하면 같은 검토 테이블에 병합됩니다.
4. 검토 테이블에서 필요한 값을 수정합니다.
5. 총수입금액을 입력해 홈택스 입력용 요약을 확인합니다.
6. 확인 필요 항목이 남아 있어도 경고를 확인한 뒤 `간편장부 엑셀 다운로드`를 진행할 수 있습니다.
7. `신고 준비 문서 다운로드`로 검토용 xlsx 초안을 만들고, 홈택스 입력 전 값을 확인합니다.

## 기술 스택

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- OpenAI Responses API
- zod
- xlsx
- exceljs
- Vercel

서버 데이터베이스, 로그인, 업로드 원본 저장은 사용하지 않습니다. API route는 업로드 파일을 일시 처리한 뒤 장부 행 또는 엑셀 파일만 반환합니다.

## 실행 방법

```bash
npm install
npm run dev
```

브라우저에서 `http://localhost:3000`을 엽니다.

품질 검증은 아래 명령으로 수행합니다.

```bash
npm run lint
npm run build
```

## 환경 변수

영수증 AI 추출을 사용하려면 `.env.local`에 다음 값을 설정합니다.

```bash
OPENAI_API_KEY=sk-...
OPENAI_RECEIPT_MODEL=gpt-4.1-mini
```

`OPENAI_API_KEY`가 없어도 앱 빌드와 UI는 동작합니다. 이 경우 영수증 분석 요청은 실패 상태가 되며, 처리 큐의 `샘플 결과 사용`으로 데모 흐름을 이어갈 수 있습니다.

## 배포

Vercel 배포 기준:

- Framework Preset: `Next.js`
- Install Command: `npm install`
- Build Command: `npm run build`
- Output Directory: 비워두기

AI 추출까지 시연하려면 Vercel Project Settings의 Environment Variables에 `OPENAI_API_KEY`와 필요 시 `OPENAI_RECEIPT_MODEL`을 등록합니다.
